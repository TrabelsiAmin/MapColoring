# Contributing to MapColoring

## How to contribute

We welcome contributions! Here are some ways you can help:

### Reporting Bugs

- Use GitHub Issues to report bugs
- Include a clear description and steps to reproduce
- Include error messages and environment details

### Suggesting Enhancements

- Open a GitHub Issue with the `enhancement` label
- Describe the desired behavior and why it would be useful
- Provide examples if applicable

### Code Contributions

1. **Fork the repository**

   ```bash
   git clone https://github.com/TrabelsiAmin/MapColoring.git
   cd MapColoring
   ```

2. **Create a feature branch**

   ```bash
   git checkout -b feature/my-feature
   ```

3. **Make your changes**
   - Follow existing code style and conventions
   - Add comments for complex logic
   - Test your changes thoroughly

4. **Add/update tests**
   - Include test cases for new features
   - Ensure existing tests still pass

5. **Commit with clear messages**

   ```bash
   git commit -m "Add feature: clear description of changes"
   ```

6. **Push to your fork and create a Pull Request**
   ```bash
   git push origin feature/my-feature
   ```

## Code Style Guidelines

- **Python**: Follow PEP 8 conventions
- **Prolog**: Use clear predicate names and add comments for complex logic
- Use meaningful variable and function names
- Keep functions small and focused

## Development Setup

```bash
# Clone the repository
git clone https://github.com/TrabelsiAmin/MapColoring.git
cd MapColoring

# Install Python dependencies
pip install -r requirements.txt

# Install SWI-Prolog (see README.md for platform-specific instructions)
```

## Testing

Before submitting:

- Test with various graph sizes
- Verify error handling with invalid inputs
- Check that visualizations render correctly

## Questions?

- Open a GitHub Discussion
- Check existing issues for similar questions
- Email the maintainer if needed
